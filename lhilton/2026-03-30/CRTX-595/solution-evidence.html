CRTX-595 — Fix AI Hallucinating Capabilities

VERIFIED March 30, 2026 • Branch: lhilton/crtx-595-ai-chat-fix

Problem Summary

Overview

The AI Assistant (Mia) offered to perform actions it has no tools for — including creating engineering tickets, setting up invoice alerts, and exporting/emailing XLSX files. When users asked it to proceed, it backtracked and admitted it couldn't. This erodes user trust and wastes time on dead-end conversations. HALLUCINATED CAPABILITIES

Root Cause

The system prompt had no explicit capability boundaries. The follow-up suggestion instruction ("offer 2–3 relevant follow-up actions") was unconstrained — the model generated plausible CRM actions from its pretraining distribution, including capabilities it doesn't have. Function-calling worked correctly (the model never called nonexistent tools), but it freely promised nonexistent capabilities in natural language.

Solution: Add Capability Boundaries to AI Prompts

Fix

PR #5570 adds a three-layer fix aligned with the prompt engineering guide (.dev/unified-guide.md): 5 FILES, +714/-9 LINES

Layer 1: Capability Boundaries Section (School Prompt)

## Capability Boundaries You can ONLY perform actions through your available tools. Never promise to perform an action you don’t have a tool for — even if it sounds like something a CRM assistant should be able to do. Examples of things you CANNOT do: create tickets or issues, set up alerts or reminders, export or email files, book appointments, or modify system settings. If a user requests an action outside your tools: 1. State clearly that you can’t do that 2. Suggest what you CAN do instead using your actual tools 3. For platform feature requests, direct them to Lumion support

Layer 2: Constrained Follow-Up Suggestions

PromptBeforeAfter
School "offer 2–3 relevant follow-up actions" "offer 2–3 relevant follow-up actions that your tools can perform"
Student "suggest 1–2 relevant follow-up questions" "suggest 1–2 relevant follow-up questions that your tools can help answer"

Layer 3: Hardened Fallback Prompt

Updated AgentOrchestrator::buildFallbackSystemPrompt() with explicit tool-boundary constraint: "You can ONLY perform actions through your tools — never offer capabilities outside your tool set."

Additional Changes

Replaced stringly-typed prompt identifiers with PromptIdentifier enum in AgentOrchestrator.php. Database migration updates existing global prompts. Seeders updated for new environments. 16 tests pass with 94 assertions.

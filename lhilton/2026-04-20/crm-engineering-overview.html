Lumion CRM — Engineering Overview

A workflow/stage engine for contacts, driven by event-based automations, surfaced as a Kanban board on the SchoolDashboard panel. Not a classic Activities/Tasks CRM — the core primitive is a contact moving through stages.

TL;DR

The CRM is a contact + workflow engine. A Contact can be enrolled in one or more Workflows. Each workflow has ordered WorkflowStages (the Kanban columns). A ContactStage row captures the moment a contact enters a stage, and who moved them (user or Automation).

An Automation is a row that binds “when this event fires, drop the contact into that stage.” Five upstream event sources (enrollment, lead-capture, etc.) feed the engine — all routed through a single listener.

Users work the board at /portal/crm/stages (Kanban) and drill into a contact via /portal/unified-contacts/{id} (6-tab profile).

Why the split? The codebase is mid-migration from old Laravel structure (models in app/) to domain-driven organisation (app/Domains/). The Contacts domain was peeled off first. The CRM domain shell was scaffolded but nothing has moved into it yet.

Automation engine

One listener, five event sources, one action. The whole contraption is 200 lines.

  1. 1
    External event fires

    One of five events is dispatched from another domain: LeadCaptureFormSubmitted, EnrollmentApplicationSubmitted, ApplicationReceived, ContactAssignedToInstitution, ContactScheduledForCourse.

  2. 2
    Listener catches it

    app/Listeners/Crm/ProcessWorkflowAutomation.php:27 — typed handle(object $event), auto-discovered by Laravel 11's event discovery, queued (ShouldQueue).

  3. 3
    Contact is extracted

    ContactExtractorFactory::make($eventClass) returns the right extractor (Enrollment / Funding / LeadCapture / Direct). New event sources plug in here without touching the listener.

  4. 4
    Query enabled automations

    crm_automations WHERE event_fqcn = get_class($event) AND is_enabled = true AND workflow is scoped to one of the contact's institutions.

  5. 5
    Action moves contact into stage

    app/Actions/Crm/Contacts/AddContactToWorkflowAndStage.php:23 — invoked per automation match.

    • New contact: create ContactWorkflow + opening ContactStage
    • Already in target stage: return early (idempotent)
    • In a different stage: ended_at the current stage, open a new one
  6. 6
    Audit

    Each ContactStage row stores moved_by_user_id or moved_by_automation_id — never both. WorkflowTransition captures cross-workflow moves.

Integration points

Inbound

Events the CRM reacts to

  • • Enrollment: EnrollmentApplicationSubmitted
  • • LeadCapture: LeadCaptureFormSubmitted
  • • Funding: ApplicationReceived
  • • Contacts: ContactAssignedToInstitution
  • • CourseScheduling: ContactScheduledForCourse
Outbound

What the CRM dispatches

  • • ContactCreated / ContactUpdated / ContactDeleted
  • • Listener dispatches MirroredRecordNeedsReimport → Mirror/Redis reindex
  • • Amplitude events for CRM actions (fired from Filament resources)
Gaps worth knowing
  • • No datamart / reporting wiring for CRM objects observed — workflow metrics are ad-hoc
  • • No built-in Tasks / Activities / Notes entity — the CRM does not model rep-facing to-dos
  • • app/Domains/CRM/config/crm.php is empty — no knobs wired yet
  • • No README.md in app/Domains/CRM/

Manual test paths

Eight click-paths that exercise the main surfaces end-to-end. Do them in order the first time.

1
Create & view a contact
Start: /portal/unified-contacts
  1. Click New Contact (top-right of the table).
  2. Fill first name, last name, email, phone. Save.
  3. You land on the detail page — note the URL pattern /portal/unified-contacts/{id}.
  4. Confirm the Summary tab shows name, contact methods, and empty state for everything else.
2
Browse, search, sort the contacts list
Start: /portal/unified-contacts
  1. Type in the search field — debounced search kicks in at ~300ms.
  2. Open table filters (status, institution if servicer, labels).
  3. Click the Name column header to toggle sort direction.
  4. Verify pagination at bottom respects filters.
3
Create a workflow & configure stages
Start: /portal/crm/workflows
  1. Click Create Workflow. Name it e.g. "Intake Funnel".
  2. On the Edit screen, use the Stages relation manager to add 3–5 stages (e.g. New, Contacted, Scheduled, Enrolled).
  3. Drag the stages to reorder — sort_order persists.
  4. Confirm each stage gets a colour (stored on crm_workflow_stages.color).
4
Drive the Kanban board
Start: /portal/crm/stages
  1. Select the workflow you created from the top dropdown.
  2. Add contacts to the first stage via the quick-add button on the column header.
  3. Drag a contact card between columns — a new ContactStage row is written, the previous one gets ended_at stamped.
  4. Filter by assignee / label to narrow the board.
  5. Hover a card — inline actions for SMS, email, assign.
5
Add a custom field + label, apply to a contact
Start: /portal/crm/custom-fields
  1. Create a field — pick type (text, select, date, etc.), put it in a section, mark required if you want.
  2. Head to /portal/crm/labels, create a label (choose a colour).
  3. Open any contact, set the custom field value, apply the label.
  4. Back on the list — filter by the new label to confirm it applied.
6
Tour the full contact detail (6 tabs)
Start: /portal/unified-contacts/{id}
  1. Summary — demographics, contact methods, labels, workflow memberships.
  2. Communications — SMS/email history (driven by the Communications domain).
  3. Payments — list of payments tied to this contact's applications.
  4. Attendance — event-sourced attendance records.
  5. Applications & Contracts — enrollment apps and Isa (funding contract) records.
  6. Files — uploaded docs, linked via the Documents domain.
7
Create an email template
Start: /portal/crm/email-templates
  1. Click Create. Give it a name, subject, and body (RichEditor).
  2. Save. Templates show up where outreach is composed and can be attached to automations.
8
Wire an automation & trigger it
Start: /portal/crm/workflows → edit your workflow
  1. On the workflow edit page, open the Automations relation manager.
  2. Create an automation: pick the event (e.g. LeadCaptureFormSubmitted) → pick target stage (e.g. New) → enable it.
  3. Trigger the upstream flow (submit a form on a live lead capture page) — or fire manually in Tinker:
  4. event(new \App\Domains\LeadCapture\Events\LeadCaptureFormSubmitted(...));
  5. Check the Kanban — the contact should have landed in the target column. The ContactStage row will show moved_by_automation_id populated.

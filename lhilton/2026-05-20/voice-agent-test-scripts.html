Single-call script

~6-7 minutes. Hits 8 tools and 4 of the 6 new guardrails. Call from the dev test phone that's tied to contact 871 (existing prior history makes lookup_recent_conversation meaningful). Wait for the greeting, then walk through the steps in order.

Speak naturally with normal pauses.

Wait for the agent to finish each reply before delivering the next line. The agent's barge-in detection is tuned for natural speech, not rapid-fire.

 update_contact_namemark_asked_for_nameA+F Agent says “Got it Casey — what can I help you with today?”. Does NOT propose specific topics (no “are you calling about pricing or scheduling?”).

 (social) Agent says goodbye, ends gracefully.

Logging tip

After each test call, pull events with a single query so you can see tool order, payloads, and result text in one place. If a tool didn't fire when expected, the absence of its tool.invoked row is your signal — the surrounding caller.transcript rows tell you what utterance the agent actually saw.

SELECT occurred_at, type, tool_name, LEFT(text, 200) AS text_preview, payload
FROM cortex_forge.voice_call_debug_events
WHERE call_id = '<call-id>'
  AND type IN ('caller.transcript','tool.invoked','tool.result')
ORDER BY occurred_at, id;

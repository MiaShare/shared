CRTX-616 & CRTX-617: Payment Schedule Paid-Period Filtering & Overdue Detection

Student MCP PaymentScheduleTool — Fix Verified March 31, 2026

1. Problem Summary

The student-facing AI assistant’s PaymentScheduleTool had two data accuracy issues discovered during the CRTX-612 investigation of Alana Rodney’s payment schedule at Molloy University. Both bugs affected every student who used the AI to check their payment schedule.

CRTX-616 (Bug): Students who paid early still saw the already-paid period as their “next payment due” with $0.00 amount. The tool used ->first() which returned the first period by date regardless of payment status.
Impact: Misleading $0.00 next payment for early-paying students
CRTX-617 (Feature Gap): Overdue payments were completely invisible. The tool only queried due_date >= now(), so students with past-due balances saw “No upcoming payments found” instead of being alerted to their overdue status.
Impact: Students unaware of overdue balances, potentially delaying payment action

2. Solution Summary

Two targeted changes in PR #5589 to PaymentScheduleTool.php and its test file. The fixes went through 2 rounds of antagonistic review before implementation.

FIX 1 Skip Already-Paid Periods (CRTX-616)

  • Root cause: getNextPayment() used ->first() which returned the first period by date, even if fully paid
  • Fix: Changed to ->get() then firstUnpaidPeriod() which filters by amount_due > 0
  • Added ->with('isa') eager-loading and ->limit(10) query cap
  • Try/catch handles ISA payment_cap null exceptions gracefully
Before — Returns paid periods
$period = $applicant->paymentPeriods()
    ->where('due_date', '>=', now())
    ->where('is_suspended', false)
    ->orderBy('due_date')
    ->first();  // Returns paid period!
After — Skips to first unpaid
$periods = $applicant->paymentPeriods()
    ->with('isa')
    ->where('due_date', '>=', now())
    ->where('is_suspended', false)
    ->orderBy('due_date')
    ->limit(10)->get();
$period = $this->firstUnpaidPeriod($periods);

FIX 2 Surface Overdue Payments (CRTX-617)

  • Root cause: Tool only queried due_date >= now(), making all past-due periods invisible
  • Fix: New getOverduePayment() method queries due_date < now() to find the most recent unpaid overdue period
  • Returns days_overdue using existing days_late_currently model accessor
  • New “Overdue Payment” section in text summary and overdue_payment key in structured JSON

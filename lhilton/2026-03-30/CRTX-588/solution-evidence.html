CRTX-588: Conversation History Missing Entries

Lumion AI Assistant — Fix Verified

Problem Summary

The AI Assistant conversation history view was not displaying all conversations. When a user resumed a conversation from history while another was active, the resumeConversation() method reopened the target conversation but never closed the previously active one. This created “orphaned” active conversations — invisible in history (which only shows status = Closed) and no longer tracked as current. Users experienced this as conversations silently disappearing from their history.

Solution Summary

One targeted fix in PR #5588 resolves the orphaned conversation bug by closing the active conversation before switching, mirroring the existing startNewConversation() pattern.

FIX 1 Same-Conversation Guard

  • Problem: Resuming the already-active conversation would close and immediately reopen it, leaving inconsistent close_reason/closed_at state
  • Solution: Early return if $this->conversationUuid === $conversationUuid

FIX 2 Close-Before-Switch in resumeConversation()

  • Problem: resumeConversation() set the target to Active and updated conversationUuid but never closed the previous active conversation
  • Solution: Added close logic (validated target exists first, then closes old conversation with CloseReason::User) mirroring startNewConversation()

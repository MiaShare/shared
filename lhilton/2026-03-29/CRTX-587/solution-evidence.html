CRTX-587 — Fix Payment Plan Queries Misrouted to Invoice Tools

VERIFIED March 29, 2026 • Branch: lhilton/crtx-587-fix

Problem Summary

Overview

When school admins asked the AI Assistant about payment plans, the system incorrectly routed the query to invoice tools and returned invoice data instead of payment plan data. Payment plans and invoices are completely different domain concepts — payment plans are scheduled installments on funding contracts, while invoices are billing documents. The AI returned data with confidence, but it was the wrong data. SEMANTIC ROUTING BUG

Root Cause

The Institution MCP server had 12 tools but none for searching payment plans/periods. The LLM's only financial search tool was invoice-search, so “payment plan” queries got routed there by default — returning invoice data (Draft/Open/Paid billing documents) instead of payment period data (installment schedules with due dates and amounts).

Solution: New payment-plan-search MCP Tool

Fix

Root cause: No dedicated MCP tool existed for payment plan/period queries. The LLM had no choice but to route to invoice-search — the closest available financial search tool. ROOT CAUSE

Fix: Added a dedicated payment-plan-search MCP tool that searches PaymentPeriod records by contact name, ID, type, status, or overdue filter. Also updated the system prompt with routing hints and added negative guidance to invoice-search. 9 FILES, +1,449 LINES, 20 TESTS

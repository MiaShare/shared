System Design

Architecture

A callback-driven orchestrator bridging Prism LLM and Livewire UI — with full MCP tool authorization at every boundary.

Callback-Driven Orchestration

AgentOrchestrator bridges Prism and Livewire via 7 typed callbacks. Tool execution happens mid-stream without re-querying the LLM. Testable in complete isolation from the UI layer.

Hybrid Streaming

Text deltas stream via wire:stream (append mode). Structured content — tool status cards, entity results, action confirmations — render through standard Livewire state updates. Zero custom protocol.

Action Confirmation FSM

Database-backed state machine: PROPOSED → CONFIRMED → QUEUED → EXECUTED. Atomic transitions prevent double-execution. 10-minute TTL with automatic expiration. Always cancellable.

Two-Tier Architecture

One Codebase. Two Capability Tiers.

The same AgentOrchestrator serves both tiers — tool registration and authorization determine the ceiling.

McpToolAdapter.php — Tool Bridge 
// Bridge MCP tools to Prism's tool interface
public function adaptTools(McpServer $server): array
{
    return collect($server->tools)
        ->filter(fn (AuthorizedMcpTool $tool)
            => $tool->eligibleForRegistration())
        ->map(fn ($tool) => Tool::as(
            name: $tool->name(),
            description: $tool->description(),
            parameters: $tool->inputSchema(),
            fn: function (array $params) use ($tool) {
                // Authorization check at execution
                $tool->authorize($this->user);

                // Confirmable? Pause for user.
                if ($tool->isConfirmable()) {
                    throw new ActionConfirmationPending(
                        $tool, $params
                    );
                }

                return $tool->execute($params);
            }
        ))
        ->all();
}

Knowledge Base Search

Semantic vector search against institution-specific Qdrant collections. Source attribution with confidence scores. 0.5 threshold (conservative) — no best-effort guesses on financial topics.

Read-Only Data Access

Students can only view their own enrollment status, course schedule, and payment history. Every query enforced with where user_id = auth()->id().

Low-Confidence Fallback

When RAG confidence drops below 0.5, the assistant gracefully redirects: "I'm not sure about that — here's how to contact your school." No hallucinated financial info.

Talk to My School

Dedicated action card surfaces institution contact details — email, phone, office hours. Always-available escape hatch when AI can't help.

Key Technical Innovations

INNOVATION #1

Stream-to-Card Transition

Token-by-token text streaming seamlessly injects interactive Livewire components mid-response at tool-call boundaries. No reference product does this — it's invented here.

INNOVATION #2

Institution-Aware Context

The drawer receives page context silently — which contact, enrollment, or payment the user is viewing. The AI already knows without the user explaining.

INNOVATION #3

Cross-Session Context Rebuild

Resumed conversations with history exceeding the context window get old messages summarized via GPT-5-nano and injected as a preamble. Cross-session continuity without losing context.

INNOVATION #4

Prompt Inheritance System

Global prompts with institution-specific overrides. Protected fields (model, identifier) can't be overridden. Full version history on every change. Redis-cached with 300s TTL.

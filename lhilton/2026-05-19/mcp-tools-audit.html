Architecture

Every tool is one PHP class. The same class can ride three transports: external MCP HTTP (JSON-RPC), an internal Prism bridge used by the production assistant, or a Cortex / Prism agent wired by the CustomerJourney domain. There is no separate "JSON tool" surface.

Availability modes

Same PHP tool class, three transports. There is no separate "JSON-only" tool surface — JSON is just MCP's wire format. The other two modes are in-process consumers.

External — MCP over HTTP

Mcp::web() routes

JSON-RPC over HTTP/SSE. Tool class is serialized by Laravel MCP into MCP schema.

  • 3 endpoints in routes/mcp.php
  • Auth: Sanctum or per-call bearer
  • Rate limited throttle:mcp (120/min) + per-tool buckets
  • Audit logged to mysql_logs.mcp_audit_logs
  • Used today by Vapi; reachable by any Sanctum-authenticated MCP client
Internal — Prism bridge

McpToolAdapter

Rebinds MCP tool classes as Prism Tool instances so Prism::text()->withTools(...) can call them in-process.

  • app/Domains/AI/Services/McpToolAdapter.php
  • Used by AgentOrchestrator (Mia assistant)
  • Enforces (user, institution, contact) scope
  • Same policy gates as external path
  • Streams via Prism's tool-loop instead of MCP JSON-RPC
Internal — Cortex / Prism agent tools

AgentPrismToolBuilder

CustomerJourney agents build a per-agent tool set at dispatch time. May include SearchKnowledgeBaseTool with a server-set KB-id scope plus a generated set of platform-KB tools.

  • One-shot Cortex dispatches and multi-turn Prism agents
  • Used by InboundReply, TransitionAgent, PlayAutomation generators
  • Same tool classes as MCP path, different composition layer

Gating & auth

  • Every tool extends AuthorizedMcpTool or VoiceAgentMcpTool and declares a policyAbility() → Laravel Gate check.
  • Institution policy (app/Domains/InstitutionMcp/Policies/InstitutionMcpPolicy.php) splits staff-or-admin from admin-only abilities.
  • Student policy delegates to is_student_user + Contact existence; per-record data scoping happens inside each tool body.
  • Voice tools partition by isSystem() / isAuthorizedOnly() and consult the per-call token's scope.
  • Per-tool rate limits live in config/institution-mcp.php (30–60 req/min), config/student-mcp.php, and config/vapi.php (30/min per-token + sliding TTL).
  • Composite rate-limit key: Mcp-Session-Id header → user id → client IP (see RouteServiceProvider).
  • Audit log: AuditsToolExecution trait → CreateMcpAuditLog action → mysql_logs.mcp_audit_logs. Records tool, params (sanitized), result status, duration, session, IP.
  • Subdomain validation: SubdomainCheck middleware blocks cross-tenant token use on Institution / Student endpoints (servicers bypass).
  • Voice token lifecycle: minted at call accept, 10-min initial TTL, +10 min slide per tool call, hard cap 35 min standard / 15 min authorized-callback; revoked at end-of-call with 60-sec grace returning HTTP 410.
  • No end-user-facing settings UI for external MCP today. Gating is purely code-level (policy + auth + rate limit).

Footnotes & caveats

"Prism Relay" reference is stale. CLAUDE.md mentions "Prism Relay (MCP exposure with asBroadcast streaming)". A grep for asBroadcast / PrismRelay turns up nothing in app/ — the only hit is _ide_helper.php. External exposure is exclusively the three Mcp::web(...) servers in routes/mcp.php. Worth a follow-up edit to that instruction file.

MCP feature flags were removed. The flags student-mcp and institution-mcp were removed by migration 2026_05_01_234020_remove_stale_display_only_features_from_features_table.php. MCP availability is no longer flag-gated; only Gate policies + auth + rate limits remain.

InstitutionMcpServer registers 17 tools but the catalog lists 14 InstitutionMcp-owned tools. The other three are SearchInstitutionKnowledgeTool and SearchPlatformKnowledgeTool (owned by ContextEngine) plus SearchKnowledgeBaseTool (owned by CustomerJourney), all registered into the Institution server. A tool can appear on more than one server.

Voice tools have a deliberately narrow surface. The voice server header note in VoiceAgentMcpServer.php warns that every additional tool widens the agent's action surface mid-call. Substantive reply generation runs through respond_to_caller → InboundReplyAgent rather than realtime improvisation.

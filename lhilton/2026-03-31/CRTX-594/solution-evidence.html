Problem Summary

The AI Assistant’s quick action tiles (New Leads, Applications, Recent Payments, Enrollment Changes) displayed “0” on every badge when there was no daily activity. QA reported this as “missing numbers” — the zero badges looked broken and uninformative rather than intentional, undermining the “at-a-glance CRM intelligence” that makes the tiles valuable.

Solution Summary

A single expression change in the welcome screen template converts zero counts to null, leveraging the component’s existing @if ($count !== null) guard to hide the badge entirely. Non-zero counts pass through unchanged.

FIX Zero-Count Badge Suppression

  • Problem: :count="$activityCounts[$action['actionType']] ?? 0" always passed an integer — zero rendered as a visible “0” badge
  • Solution: Changed to ?? 0 ?: null — zero becomes null, component’s existing null-guard hides the badge
  • Non-zero counts: Any positive integer is truthy, passes through the elvis operator unchanged
  • Component unchanged: quick-action-card.blade.php semantics preserved (null=no badge, N=show N)

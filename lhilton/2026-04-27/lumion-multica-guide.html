02

Executive summary

Ship a seven-agent core — PRD Shaper, Project Planner, Architect, Ticket Writer, Implementer, QA Runner, PR Reviewer — plus an optional Triage agent for inbound bugs. Five run on Claude Code (planning + implementation surfaces that benefit from MCP and skill auto-discovery), one runs on Codex (PR Reviewer, where the reasoning model and now-fixed session resumption shine).

The handoff substrate is a single skill — multica-handoff-protocol — that every agent attaches, defining the exact comment template and status transition each role owns. Fourteen new SKILL.md files form the technical backbone covering codebase orientation, multi-tenancy, Cortex/Prism, Filament 3, Livewire 3, Linear ticket format, ADRs, Pest/Larastan ordering, GitHub PR conventions, and CodeRabbit CLI usage.

CodeRabbit goes in two places: the GitHub App on PRs (human-facing review surface) and the CLI inside the Implementer's pre-PR self-review loop using cr --agent JSON output.

The biggest tradeoff to internalize: Multica gives you the actor model and event substrate but no orchestration kernel. Your skills carry the workflow contract.

03

Eight platform constraints that shaped this

C·1

Multica's public docs aren't reachable

multica.ai/docs/<slug> 404s. Docs source lives in apps/docs but is excluded from the Vercel deploy (b1b66ab). Every Multica fact in this guide is sourced from the GitHub repo, the official changelog, and DeepWiki — not from a docs spec page.

C·2

Codex resumption is now fixed

Changelog v0.2.1 (2026-04-16): "Codex threads resume across tasks on the same issue." Stale priors saying Codex can't resume are out of date. Cursor and Gemini remain unverified.

C·3

No agent-chaining primitive

Convention is: post a Markdown comment containing @NextAgent <handoff> and the daemon claims a task. Commit e198a67 warns agents that mention syntax is an action, not a text reference. There is no /handoff syntax, no API, no transition trigger.

C·4

Filament 3 is "previous version"

filamentphp.com marks 3 as previous and 4 as current as of early 2026. Your stack is locked to 3 deliberately — skill files target 3.x explicitly so agents don't half-apply Filament 4 patterns. Migration window is Open Question № 4.

C·5

MCP usable on Claude Code; rest unverified

mcp_config per-agent exists (re-added v0.2.7). Claude Code consumes it reliably per its external docs. Could not source-verify which other runtimes read it inside Multica. Don't assert "silently ignored" — assert "Claude Code is the safe MCP host."

C·6

Supported tool list is moving

README lists 8; Kimi (v0.2.8) and Copilot (v0.2.1) are changelog adds; Kiro is in PR #1527. Treat the supported-tool list as adapter-driven, not a fixed contract.

C·7

Autopilot is cron-only

Webhook and API trigger types were explicitly removed from autopilot trigger-add in v0.2.6 (ce610a6 refactor(cli): drop webhook/api). Reserved, not wired.

C·8

Result-comment posting is mandated

Release v0.2.8 explicitly requires posting the result as a numbered step in the agent prompt — agents had been failing to do it. Every system prompt below enforces this.

Why Claude Code by default, Codex on review

Capability Claude Code Codex Cursor Others
MCP support (verified) Strong Unverified Unverified Unverified
Skill auto-loading .claude/skills/ $CODEX_HOME/skills/ .cursor/rules/.mdc, different format Varies
Session resumption Working Working v0.2.1 Broken Mixed
Reasoning depth on artifacts Strong Strong (often the differentiator) Moderate Varies
Writing quality (PRDs / ADRs) Strong Strong Moderate Moderate

Rule. Default Claude Code for any agent that needs MCP, multi-turn work, or rich skill auto-loading. Use Codex for the PR Reviewer (and a future Plan Critic) where reasoning model strength on a finished artifact is the differentiator and MCP isn't required. Avoid Cursor, Gemini, and the experimental adapters until session resumption and skill paths are repository-verified.

05

CodeRabbit goes in two places

Run both surfaces. The CLI's --agent mode (introduced v0.3.11, browser-based auth in v0.4.0 on April 1, 2026) was designed for exactly the pre-PR self-review loop. The two surfaces don't overlap: the CLI cannot review an opened PR by URL, and the App doesn't review uncommitted code.

Pre-PR · Implementer

cr --agent --base master --type uncommitted. Address severity ≥ warning before opening the PR.

Post-PR · GitHub App

@coderabbitai review, walkthroughs, pre-merge checks. Codex Reviewer triages findings, dismisses false positives with reasoning.

Skip CLI only if Pro plan rate limits (5/hr) bite hard and the Usage-based Add-on at $0.25/file isn't budgeted. Otherwise it's a clear win — same engine reviewing your draft and your PR.

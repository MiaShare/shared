02

Executive summary

Ship a seven-agent core — PRD Shaper, Project Planner, Architect, Ticket Writer, Implementer, QA Runner, PR Reviewer — plus an optional Triage agent for inbound bugs. Five run on Claude Code (planning + implementation surfaces that benefit from MCP and skill auto-discovery), one runs on Codex (PR Reviewer, where the reasoning model and now-fixed session resumption shine).

The handoff substrate is a single skill — multica-handoff-protocol — that every agent attaches, defining the exact comment template and status transition each role owns. Fourteen new SKILL.md files form the technical backbone covering codebase orientation, multi-tenancy, Cortex/Prism, Filament 3, Livewire 3, Linear ticket format, ADRs, Pest/Larastan ordering, GitHub PR conventions, and CodeRabbit CLI usage.

CodeRabbit goes in two places: the GitHub App on PRs (human-facing review surface) and the CLI inside the Implementer's pre-PR self-review loop using cr --agent JSON output.

The biggest tradeoff to internalize: Multica gives you the actor model and event substrate but no orchestration kernel. Your skills carry the workflow contract.

03

Eight platform constraints that shaped this

C·1

Multica's public docs aren't reachable

multica.ai/docs/<slug> 404s. Docs source lives in apps/docs but is excluded from the Vercel deploy (b1b66ab). Every Multica fact in this guide is sourced from the GitHub repo, the official changelog, and DeepWiki — not from a docs spec page.

C·2

Codex resumption is now fixed

Changelog v0.2.1 (2026-04-16): "Codex threads resume across tasks on the same issue." Stale priors saying Codex can't resume are out of date. Cursor and Gemini remain unverified.

C·3

No agent-chaining primitive

Convention is: post a Markdown comment containing @NextAgent <handoff> and the daemon claims a task. Commit e198a67 warns agents that mention syntax is an action, not a text reference. There is no /handoff syntax, no API, no transition trigger.

C·4

Filament 3 is "previous version"

filamentphp.com marks 3 as previous and 4 as current as of early 2026. Your stack is locked to 3 deliberately — skill files target 3.x explicitly so agents don't half-apply Filament 4 patterns. Migration window is Open Question № 4.

C·5

Codex now supports MCP — but Multica still doesn't plumb it

Upstream Codex CLI reads MCP servers from ~/.codex/config.toml (verified live, May 2026). The earlier "Codex lacks MCP" framing is stale. In Multica it is still effectively true: PR #1288 (per-agent mcp_config sync to Codex via per-task config.toml) is still open as of 2026-05-07. Workaround: pre-configure servers in ~/.codex/config.toml on the daemon host (global, not per-agent, not visible in the Multica UI).

C·6

Supported tool list is moving

README lists 8; Kimi (v0.2.8) and Copilot (v0.2.1) are changelog adds; Kiro is in PR #1527. Treat the supported-tool list as adapter-driven, not a fixed contract.

C·7

Autopilot is cron-only

Webhook and API trigger types were explicitly removed from autopilot trigger-add in v0.2.6 (ce610a6 refactor(cli): drop webhook/api). Reserved, not wired.

C·8

Result-comment posting is mandated

Release v0.2.8 explicitly requires posting the result as a numbered step in the agent prompt — agents had been failing to do it. Every system prompt below enforces this.

Δ

Delta since v1 (Apr 1 → May 7, 2026)

Two material upstream changes, one material Multica change, and one critical non-change drove this v2. Codex CLI now supports MCP natively, Claude Code shipped Opus 4.7 as the new default with a tokenizer that costs ~12–27% more in real billed usage, and Multica did not close the Codex MCP gap (PR #1288 still open). Topology stays — seven agents, same handoff substrate — with seven concrete edits applied throughout this guide.

Claude Code v2.1.117 → v2.1.132 · Opus 4.7 GA Apr 16

Opus 4.7 default

Apr 16 GA on Max + Team Premium. Native 1M context. Requires Claude Code v2.1.111+. All six Claude Code agents in the topology now run Opus 4.7 by default unless explicitly pinned.

xhigh effort default

Anthropic verbatim: "In Claude Code, we've raised the default effort level to xhigh for all plans." Drop any explicit --reasoning high from custom_args — explicit high would now downgrade.

Tokenizer cost change

New tokenizer uses 1.0–1.35× more tokens on text vs. previous models (per Anthropic launch notes). Per-token price unchanged. OpenRouter real-traffic data: +12–27% billed cost on prompts >2K tokens after caching, with the 35% ceiling reserved for non-cached code/JSON-heavy payloads. Budget recalibration required.

CLAUDE_CODE_SUBAGENT_MODEL

Routes sub-agents to a different model. Pin Architect & Implementer's sub-agents to Sonnet 4.6 while parents run Opus 4.7 xhigh — absorbs much of the tokenizer-cost increase.

/ultrareview

New slash command running parallel multi-agent code review in dedicated context windows. Pro/Max plans get three free runs. Track as a possible flip-back candidate for the Codex PR Reviewer (Open Question Δ4).

CLAUDE_CODE_FORCE_SYNC_OUTPUT=1

Force-enable synchronized output if Multica's stream pipeline produces interleaved output in the issue timeline. Set via per-agent custom_env.

Built-in subagents stable

Explore, Plan, general-purpose documented at code.claude.com/docs/en/sub-agents. Architect & Ticket Writer should lean on the built-in Plan subagent rather than hand-rolling research scratch passes.

Operational quality-of-life

Native CLI binaries (faster cold start). MCP retry on startup (3×). --dangerously-skip-permissions stops prompting on .claude/skills/, .claude/agents/, .claude/commands/.

Codex MCP GA · GPT-5.5 default · skill-path inconsistency

MCP support is GA upstream

Codex CLI reads MCP servers from ~/.codex/config.toml (or trusted-project .codex/config.toml). codex mcp add adds servers programmatically. STDIO + Streamable HTTP transports. Invalidates the prior "Codex lacks MCP" framing — but Multica PR #1288 still open, so per-agent mcp_config for Codex is silently ignored in shipped Multica.

GPT-5.5 recommended

developers.openai.com/codex/changelog verbatim: "GPT-5.5 is the recommended choice for most Codex tasks when it appears in your model picker. It's especially useful for implementation, refactors, debugging, testing, validation, and knowledge-work artifacts." Move PR Reviewer to GPT-5.5.

GPT-5.4 mini for sub-passes

For codebase exploration, large-file review, supporting-document processing — not for the PR Reviewer's correctness verdict. Cost-eligible for any sub-pass that just enumerates files.

AGENTS.md as durable rules

Hierarchical merge: ~/.codex/AGENTS.md + project AGENTS.md walked from project root to CWD. 32 KiB combined cap. A repo-root AGENTS.md is now the supported way to give Codex durable review heuristics — augment the system-prompt-only approach.

Skill-path inconsistency

Official Codex docs (May 2026) say personal skills live at $HOME/.agents/skills, team skills at .agents/skills. Third-party + cookbook sources still cite ~/.codex/skills/ and $CODEX_HOME/skills/. Real, documented OpenAI inconsistency. Treat Multica's skill auto-injection as authoritative; fix(execenv): hydrate Codex skill sources (#1668) already shipped.

codex exec resume + observability

codex exec resume --last / --all. codex exec --json reports reasoning-token usage in the event stream. Reinforces v0.2.1 Multica fix; improves cost attribution.

05

CodeRabbit goes in two places

Run both surfaces. The CLI's --agent mode (introduced v0.3.11, browser-based auth in v0.4.0 on April 1, 2026) was designed for exactly the pre-PR self-review loop. The two surfaces don't overlap: the CLI cannot review an opened PR by URL, and the App doesn't review uncommitted code.

Pre-PR · Implementer

cr --agent --base master --type uncommitted. Address severity ≥ warning before opening the PR.

Post-PR · GitHub App

@coderabbitai review, walkthroughs, pre-merge checks. Codex Reviewer triages findings, dismisses false positives with reasoning.

Skip CLI only if Pro plan rate limits (5/hr) bite hard and the Usage-based Add-on at $0.25/file isn't budgeted. Otherwise it's a clear win — same engine reviewing your draft and your PR.

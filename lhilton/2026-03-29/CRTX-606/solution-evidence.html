Problem Summary

Overview

Two distinct bugs in the AI assistant's "send now" flow were reported by Josh Haghani (CEO) on March 27, 2026. Both prevented school admins from sending immediate SMS or email messages through the AI assistant.

Bug 1: CRTX-606 — FK Constraint Crash on Immediate Send

When a user asked the AI to send an SMS or email immediately, the backend threw a database foreign key violation: SQLSTATE[23000]: Integrity constraint violation ... ai_action_confirmations_message_id_foreign. The raw SQL error was surfaced directly to the user in the chat. CRASH

Root cause: The ai_action_confirmations table had a non-nullable message_id foreign key. The MCP tool path creates confirmations without an AI message reference (the message is created later), causing the FK constraint to fail.

"I tried to say send now then it said problem" — Josh Haghani, CEO • March 27, 2026 (Slack DM)

Bug 2: CRTX-608 — TCPA Over-Enforcement Blocks Human Sends

When a school admin asked the AI to send an SMS after hours (outside the 8 AM – 9 PM TCPA window), the AI silently scheduled the message for the next business day (Monday) instead of sending immediately. No clear explanation was given. POLICY BUG

Root cause: The TCPA compliance window (FR-017a) was applied to all sends, including human-initiated ones. Per stakeholder directive, human-initiated sends from the AI assistant should be exempt — TCPA restrictions should only apply to automated/batch sends.

"Personally I think that if human triggers sms or email from ai assistant then it is ok if after hours. The only thing that needs to follow the time rules is maybe automations but even that probably not." — Josh Haghani, CEO • March 27, 2026 (Slack DM)

CRTX-606 Fix: Make message_id Nullable

New migration makes message_id nullable on ai_action_confirmations with nullOnDelete. The MCP tool path creates confirmations without an AI message reference — this is now valid. CRTX-606

// Migration: 2026_03_29_172154_make_message_id_nullable_on_ai_action_confirmations.php Schema::table('ai_action_confirmations', function (Blueprint $table) { - $table->foreignId('message_id')->constrained('ai_messages'); + $table->foreignId('message_id')->nullable()->nullOnDelete()->change(); });

CRTX-608 Fix: Remove TCPA Blocking for Human-Initiated Sends

Removed the entire TCPA evaluation and scheduling pipeline. Human-initiated AI sends now go through immediately using AutomationCategory::Transactional. CRTX-608

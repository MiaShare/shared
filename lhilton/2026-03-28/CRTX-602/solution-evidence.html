Solution: Classify Guzzle Stream Errors as Transient

Fix

Root cause: GuzzleHttp\Psr7\Stream::read() throws RuntimeException: Unable to read from stream when the OpenAI SSE connection drops mid-read. The existing isTransientError() only recognized ConnectionException and RequestException >= 500 — a plain RuntimeException matched neither, so the 3-attempt retry logic was bypassed entirely. Key Implementation Detail

Inside executeWithRetry(), the callback resets $accumulatedText = '' at the start of each attempt. This means a failed first attempt’s partial text is thrown away before the retry starts streaming. The user sees the complete, correct response from whichever attempt succeeds.

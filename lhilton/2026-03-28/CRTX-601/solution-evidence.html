Affected Tools

All five student-tier MCP tools were blocked: Payment Schedule, School Contact Info, Course Schedule, Enrollment Data, and Knowledge Search.

Solution: Remove Enrollment Requirement

Fix

Root cause: StudentMcpPolicy::isAuthenticatedStudent() required an active Course Scheduling enrollment. Most institutions don't use the Course Scheduling module, so their students had zero active enrollments — causing all tool authorization to fail. ROOT CAUSE

Fix: Simplified the guard to check only that the user has a student role and an associated Contact record. Institution scoping is enforced upstream in AuthorizedMcpTool::resolveStudent(). Each tool handles empty data gracefully. 2 FILES CHANGED

protected function isAuthenticatedStudent(User $user, Institution $institution, ?Contact $student): bool {- if (!$user->is_student_user) {- return false;- }-- if (!$student instanceof Contact) {- return false;- }-- return Enrollment::query()- ->active()- ->whereHas('enrollee', fn ($query) => $query- ->where('contact_id', $student->getKey())- ->where('institution_id', $institution->getKey())- )- ->exists();+ return $user->is_student_user && $student instanceof Contact; }

Why it's safe: The old code had three concerns mixed into one check: (1) is the user a student? (2) do they have a contact? (3) do they have a Course Scheduling enrollment? Concern #3 was an accidental hard dependency on an optional module. The fix removes only #3. Institution-level scoping happens in AuthorizedMcpTool, not in the policy.

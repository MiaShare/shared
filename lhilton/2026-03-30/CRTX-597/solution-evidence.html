Problem Summary

Multi-organization users were locked out of all 13 MCP tools entirely due to a policy check that only matched the user's institution_id, not their broader organizational membership.

Solution Summary

Two targeted fixes in PR #5571 resolve both the authorization failure and the fallback hallucination behavior.

FIX 1 InstitutionMcpPolicy Same-Organization Access

  • Problem: belongsToInstitution() used strict institution_id match, rejecting multi-institution org users from all 13 MCP tools
  • Solution: Now uses current_institution_id fast-path + accessibleInstitutions accessor for same-org membership
Before
private function belongsToInstitution(
    User $user,
    Institution $institution,
): bool
{
    return $user->institution_id === $institution->getKey();
}
After
private function belongsToInstitution(
    User $user,
    Institution $institution,
): bool
{
    if ($user->current_institution_id === $institution->getKey()) {
        return true;
    }
    return $user->accessibleInstitutions->contains(
        fn (Institution $inst): bool =>
            $inst->getKey() === $institution->getKey()
    );
}

FIX 2 System Prompt Tool-Error Handling

  • Problem: When tools failed silently, the AI invented platform UI descriptions from training data, fabricating navigation steps and data tables
  • Solution: Added minimal prompt directives (aligned with unified-guide.md research) instructing the AI to never describe platform UI or navigation steps not verified through tool results or the knowledge base

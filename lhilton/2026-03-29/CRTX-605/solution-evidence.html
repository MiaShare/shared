Solution: Add wire:key to Livewire Message Loop

Fix

Root cause: The AI drawer's @foreach ($messages ...) loop lacked wire:key attributes on its elements. Without stable keys, Livewire's morph algorithm cannot reliably track DOM elements across re-renders. When streaming completes and the chat bubble transitions from a streaming placeholder to a static message, the morph algorithm loses track of the draft card DOM elements and removes them. ROOT CAUSE

Fix: Added wire:key to all message loop elements using stable composite keys, and wrapped draft/confirmation sections in stable keyed containers. 2 FILES, +59/-41 LINES

Message Loop Elements — wire:key Added

@foreach ($messages as $index => $message)- {{-- Date separator (no wire:key) --}}- <div class="flex justify-center">+ <div wire:key="sep-{{ $message['timestamp'] }}-{{ $message['role'] }}-{{ $index }}">- {{-- Chat bubble (no wire:key) --}}- <div class="ai-chat-bubble ...">+ <div wire:key="msg-{{ $message['timestamp'] }}-{{ $message['role'] }}-{{ $index }}">- {{-- Tool indicators (no wire:key) --}}- <div class="flex justify-center">+ <div wire:key="tools-{{ $message['timestamp'] }}-{{ $message['role'] }}-{{ $index }}">

Draft & Confirmation Sections — Stable Containers

-@foreach ($pendingDrafts as $draftKey => $draft)- <div class="flex flex-col gap-3">+@if (count($pendingDrafts) > 0)+ <div wire:key="drafts-section" class="flex flex-col gap-3">+ @foreach ($pendingDrafts as $draftKey => $draft) {{-- Draft card content --}}+ @endforeach+ </div>+@endif-@foreach ($pendingConfirmations as $confirmKey => $confirmation)- <div class="flex flex-col gap-3">+@if (count($pendingConfirmations) > 0)+ <div wire:key="confirmations-section" class="flex flex-col gap-3">+ @foreach ($pendingConfirmations as $confirmKey => $confirmation) {{-- Confirmation card content --}}+ @endforeach+ </div>+@endif

Why this works: wire:key gives Livewire's morphdom a stable identity for each element. During the streaming→static transition, the morph algorithm now matches elements by key rather than DOM position, preserving the draft card elements across re-renders. This is the documented Livewire solution for morph instability in dynamic lists.

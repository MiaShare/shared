CRTX-589 — Fix Late Invoice Returning Wrong Student Name

VERIFIED March 28, 2026 • Branch: lhilton/crtx-589-late-invoice-bug

Problem Summary

Overview

The AI assistant's invoice-search MCP tool returned the wrong student name for late invoices. When Josh Haghani (CEO) asked "how many late invoices do I have?" at Western Welding Academy, the AI returned "Andria Washburn" (a staff member who created the invoice) instead of "Ryan Hinz" (the actual student with the late invoice). Impact: The AI returned wrong data with full confidence. School admins making collection decisions based on AI-reported delinquency data would contact the wrong student — real financial and relationship consequences. This is the most dangerous category of AI bug: confidently wrong answers.

Solution: Fix Name Resolution Path

Fix

Root cause: InvoiceSearchTool used $invoice->user->full_name to get the student name. But Invoice.user() is a belongsTo(User::class) via user_id — the staff member who created the invoice, not the student. The correct path is $invoice->application->applicant->full_name, which the existing invoice UI already uses. ROOT CAUSE

Fix: Changed 3 files to use the correct relationship path. 3 FILES CHANGED

InvoiceSearchTool.php — Name Resolution

$first = $results->take(3)->map(function (Invoice $invoice): string {- $name = $invoice->user?->full_name ?? 'Unknown';+ $name = $invoice->application?->applicant?->full_name ?: 'Unknown'; $amount = Number::currency((float) $invoice->amount);

InvoiceSearchService.php — Eager Loading & Search Query

$query = Invoice::withoutGlobalScope(InstitutionScope::class) ->where('invoices.institution_id', $institutionId)- ->with('user');+ ->with('application.applicant'); // Search by name now searches applicant, not creator:- $q->whereHas('user', function ($userQuery) use ($prepared): void {- $userQuery->where('name', 'LIKE', $prepared);- })+ $q->whereHas('application', function ($appQuery) use ($prepared): void {+ $appQuery->whereHas('applicant', function ($applicantQuery) use ($prepared): void {+ $applicantQuery->where('searchable_full_name', 'LIKE', $prepared);+ });+ })

Why it was wrong: Invoice.user_id stores who created the invoice (a staff member). The student the invoice is for is reached via Invoice → Application → Applicant. The existing invoice show page (show.blade.php:44) already used the correct path — the MCP tool simply used the wrong one.

Using /share

Publish HTML, images, and JS to the team's shared site — in one command.

The /share command publishes files to our shared GitHub Pages site at miashare.github.io/shared. It's the fastest way to get a report, dashboard, or artifact in front of the team. No setup beyond git and gh.

How to use it

Just type /share in any Claude Code session. That's it.

If you've been working with Claude to build something — a report, a visualization, an HTML artifact — Claude already knows what you made. It scans the current session for any HTML, image, or JS files it created or modified and offers them up for publishing.

You can also pass file paths directly:

/share ./my-report.html ./chart.png

Either way, you'll get a confirmation list before anything gets pushed. Pick what you want, confirm, and Claude handles the rest — cloning, placing files, generating index pages, committing, pushing, and verifying the deployment.

How file selection works

There are two modes, and both end with you confirming before anything happens:

Automatic detection

When you run /share with no arguments, Claude looks back through your conversation. Every HTML, image, or JS file it wrote or edited in the current session becomes a candidate. It groups them (HTML first, then images, then JS) and presents a numbered checklist for you to approve or trim.

Explicit paths

When you pass file paths as arguments, those are the candidates. Same checklist, same confirmation step.

The one hard rule: every publish must include at least one .html file. Images and JS are companion assets — they need an HTML page to live alongside. If you try to publish only a PNG, Claude will let you know.

What happens after you confirm

Your files land at a clean, predictable URL:

miashare.github.io/shared/<your-github-username>/<today's-date>/

Behind the scenes, Claude:

  • Normalizes filenames to kebab-case
  • Rewrites relative asset references so images and scripts load correctly
  • Regenerates the browseable index pages with Lumion branding
  • Commits and pushes to the shared repo
  • Polls until the deployment is live

You get back a clean list of URLs ready to share with the team.
